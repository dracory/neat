Query Builder

This document describes the query builder API in Neat ORM.

Basic Queries

Selecting Records

// Get all records
var users []User
err := db.Query().Get(&users)

// Get first record
var user User
err := db.Query().First(&user)

// Find by ID
var user User
err := db.Query().Find(&user, 1)

Where Clauses

// Laravel-style where (implicit = operator)
db.Query().Where("name", "John")

// Multiple conditions
db.Query().Where("name", "John").Where("age", 30)

// Explicit operator with spaces
db.Query().Where("age > ?", 18)

// Explicit operator without spaces
db.Query().Where("age>?", 18)

// OrWhere (also supports Laravel-style)
db.Query().Where("name", "John").OrWhere("name", "Jane")

// Complex conditions
db.Query().Where("name LIKE ?", "John%")
db.Query().Where("age BETWEEN ? AND ?", 18, 65)
db.Query().Where("id IN (?)", []any{1, 2, 3})

Where Syntax Options:

The query builder supports multiple Where syntax styles:

  1. Laravel-style (implicit = operator):
    • Where("column", "value") → column = ?
    • OrWhere("column", "value") → column = ?
  2. Explicit operator with spaces:
    • Where("column = ?", "value")
    • Where("age > ?", 18)
    • Where("name LIKE ?", "pattern")
  3. Explicit operator without spaces:
    • Where("column=?", "value")
    • Where("age>?", 18)
    • Where("name LIKE ?", "pattern")

The Laravel-style syntax is automatically applied when:

  • Exactly one argument is provided
  • The query string does not contain an SQL operator
  • The query string does not contain operator-like keywords (LIKE, IN, BETWEEN, etc.)

Ordering

// Order by
db.Query().OrderBy("created_at", "desc")

// Multiple order by
db.Query().OrderBy("name", "asc").OrderBy("created_at", "desc")

Limit and Offset

// Limit
db.Query().Limit(10)

// Offset
db.Query().Offset(20)

// Pagination
db.Query().Limit(10).Offset(20)

Aggregations

// Count
var count int64
err := db.Query().Count(&count)

// Sum
var sum float64
err := db.Query().Sum("amount", &sum)

// Average
var avg float64
err := db.Query().Avg("amount", &avg)

// Min
var min float64
err := db.Query().Min("amount", &min)

// Max
var max float64
err := db.Query().Max("amount", &max)

// Exists
var exists bool
err := db.Query().Where("id", 1).Exists(&exists)

Sugar Methods (AsVar)

Neat ORM provides convenience methods called "sugar methods" that return values directly instead of requiring pointer parameters. These methods use the AsVar suffix and provide a more idiomatic Go API while maintaining the existing performance-oriented API.

Base API vs Sugar API

// Base API (performance-oriented, no allocation)
var count int64
err := db.Query().Count(&count)

// Sugar API (usability-oriented, minor allocation)
count, err := db.Query().CountAsVar()

Aggregation Sugar Methods

// CountAsVar
count, err := db.Query().CountAsVar()

// SumAsVar
total, err := db.Query().SumAsVar("amount")

// AvgAsVar
average, err := db.Query().AvgAsVar("amount")

// MinAsVar
minimum, err := db.Query().MinAsVar("amount")

// MaxAsVar
maximum, err := db.Query().MaxAsVar("amount")

// ExistsAsVar
exists, err := db.Query().Where("id", 1).ExistsAsVar()

Selection Sugar Methods

// PluckAsVar - get single column values
emailsAny, err := db.Query().PluckAsVar("email")
emails := emailsAny.([]string)

// ValueAsVar - get single column value from first record
emailAny, err := db.Query().Where("id", 1).ValueAsVar("email")
email := emailAny.(string)

Retrieval Sugar Methods

// FirstAsVar - get first record
userAny, err := db.Query().FirstAsVar()
user := userAny.(User)

// FindOneAsVar - alias for FirstAsVar (Sequelize-style)
userAny, err := db.Query().FindOneAsVar()
user := userAny.(User)

// GetAsVar - get all records
usersAny, err := db.Query().GetAsVar()
users := usersAny.([]User)

// AllAsVar - alias for GetAsVar (Django-style)
usersAny, err := db.Query().AllAsVar()
users := usersAny.([]User)

// FindAllAsVar - alias for GetAsVar (Sequelize-style)
usersAny, err := db.Query().FindAllAsVar()
users := usersAny.([]User)

// FindAsVar - find with conditions
usersAny, err := db.Query().FindAsVar("age > ?", 18)
users := usersAny.([]User)

When to Use Sugar Methods:

  • Application logic and business code
  • API handlers and controllers
  • Readability matters more than micro-optimizations
  • One-off queries and scripts
  • Development and prototyping

When to Use Base Methods:

  • Performance-critical code paths
  • Hot loops and high-frequency operations
  • Writing to existing variables
  • Avoiding allocations is important

Type Safety:

Since sugar methods return any or []any, you'll need to type-assert the results:

// For single records
userAny, err := db.Query().FirstAsVar()
if err != nil {
    return err
}
user := userAny.(User)

// For slices
usersAny, err := db.Query().GetAsVar()
if err != nil {
    return err
}
users := usersAny.([]User)

Performance Impact:

Sugar methods introduce minimal overhead:

  • Aggregation methods: ~1-2 ns overhead (negligible)
  • Single record methods: ~5-10 ns overhead (negligible)
  • Slice methods: ~10-20 ns overhead (negligible)

For most applications, this overhead is insignificant. Only consider using base methods in performance-critical hot paths.

Selection Methods

// Select specific columns
db.Query().Select("id", "name", "email")

// Select with subquery
subquery := db.Query().Select("user_id").From("orders")
db.Query().Select(subquery)

// Distinct results
db.Query().Distinct("category")

// Pluck single column
var names []string
err := db.Query().Pluck("name", &names)

// Get single value
var count int
err := db.Query().Value("COUNT(*)", &count)

// Scan raw results
var results []map[string]any
err := db.Query().Raw("SELECT * FROM users").Scan(&results)

Advanced Retrieval Methods

// All - Django-style alias for Get
var users []User
err := db.Query().All(&users)

// FindAll - Sequelize-style alias for All
err := db.Query().FindAll(&users)

// FindOne - Sequelize-style alias for First
var user User
err := db.Query().FindOne(&user)

// Find with conditions
var users []User
err := db.Query().Find(&users, "active = ?", true)

// FirstOrFail - error if not found
var user User
err := db.Query().Where("email", "john@example.com").FirstOrFail(&user)

// FindOrFail - error if not found
var user User
err := db.Query().FindOrFail(&user, 1)

// FirstOr - execute callback if not found
var user User
err := db.Query().Where("email", "john@example.com").FirstOr(&user, func() error {
    user = User{Name: "Default User"}
    return nil
})

// FirstOrCreate - get or create
var user User
err := db.Query().Where("email", "john@example.com").FirstOrCreate(&user)

// FirstOrNew - get or new instance (not saved)
var user User
err := db.Query().Where("email", "john@example.com").FirstOrNew(&user)

// WhereNotIn

// OrWhereIn

// OrWhereNotIn
db.Query().Where("name", "John").OrWhereNotIn("id", []any{1, 2, 3})

// WhereNotBetween
db.Query().WhereNotBetween("age", 18, 65)

// OrWhereBetween
db.Query().Where("name", "John").OrWhereBetween("age", 18, 25)

// OrWhereNotBetween
db.Query().Where("name", "John").OrWhereNotBetween("age", 30, 65)

// WhereNotNull
db.Query().WhereNotNull("email")

// OrWhereNull
db.Query().Where("name", "John").OrWhereNull("deleted_at")

// WhereColumn - compare columns
db.Query().WhereColumn("created_at", "<", "updated_at")

// OrWhereColumn
db.Query().Where("name", "John").OrWhereColumn("age", ">", "18")

// WhereExists - subquery

db.Query().WhereExists(subquery)

// WhereNot
db.Query().WhereNot("active", true)

// Exclude - Django-style alias for WhereNot
db.Query().Exclude("active", true)

// OrWhereNot
db.Query().Where("name", "John").OrWhereNot("active", true)

db.Query().WhereAll([]string{"first_name", "last_name"}, "LIKE", "%John%")

// WhereNone - NOT OR across multiple columns
db.Query().WhereNone([]string{"first_name", "last_name"}, "LIKE", "%John%")

Advanced Joins

// Left join

Soft Delete Methods

// Include soft-deleted records
db.Query().WithSoftDeleted().Get(&users)

// Deprecated alias
db.Query().WithTrashed().Get(&users)

// Only soft-deleted records
db.Query().OnlySoftDeleted().Get(&users)

// Deprecated alias
db.Query().OnlyTrashed().Get(&users)

// Exclude soft-deleted (default)
db.Query().WithoutSoftDeleted().Get(&users)

// Deprecated alias
db.Query().WithoutTrashed().Get(&users)

// Restore soft-deleted records

// Deprecated alias
result, err := db.Query().Where("id", 1).Restore()

Relationship Methods

// Eager load relationships
db.Query().With("posts").Get(&users)

// Eager load multiple relationships
db.Query().With("posts", "comments").Get(&users)

// Exclude relationships
db.Query().Without("posts").Get(&users)

// Load relationship for model
var user User
db.Query().First(&user)
db.Query().Load(&user, "posts")

// Load missing relationships
db.Query().LoadMissing(&user, "posts")

// Eager load relationship count
db.Query().WithCount("posts").Get(&users)

// Filter by relationship existence
db.Query().WithExists("posts").Get(&users)

// Get association instance
assoc := db.Query().Model(&user).Association("posts")

Chunking and Cursor

// Process records in chunks
err := db.Query().Chunk(100, func(users []User) error {
    // Process each chunk
    for _, user := range users {
        fmt.Println(user.Name)
    }
    return nil
})

// Use cursor for efficient iteration
cursor, err := db.Query().Cursor()
if err != nil {
    return err
}
defer cursor.Close()

for cursor.Next() {
    var user User
    if err := cursor.Scan(&user); err != nil {
        return err
    }
    fmt.Println(user.Name)
}

Transaction Methods

// Execute within transaction
err := db.Query().Transaction(func(tx Query) error {
    // Perform operations
    if err := tx.Create(&user); err != nil {
        return err // Will rollback
    }
    return nil // Will commit
})

// Manual transaction control
tx, err := db.Query().Begin()
if err != nil {
    return err
}

if err := tx.Create(&user); err != nil {
    tx.Rollback()
    return err
}

if err := tx.Commit(); err != nil {
    return err
}

// Savepoints
err := db.Query().Transaction(func(tx Query) error {
    if err := tx.SavePoint("before_update"); err != nil {
        return err
    }
    
    if err := tx.Update(&user); err != nil {
        tx.RollbackTo("before_update")
        return err
    }
    
    return nil
})

// Transaction callbacks
err := db.Query().
    BeforeCommit(func() error {
        fmt.Println("About to commit")
        return nil
    }).
    AfterCommit(func() error {
        fmt.Println("Committed successfully")
        return nil
    }).
    Transaction(func(tx Query) error {
        return tx.Create(&user)
    })

// Check if in transaction
if db.Query().InTransaction() {
    fmt.Println("Currently in transaction")
}

Advanced Operations

// Increment column
err := db.Query().Where("id", 1).Increment("views")

// Increment by amount
err := db.Query().Where("id", 1).Increment("views", 5)

// Decrement column
err := db.Query().Where("id", 1).Decrement("stock")

// Decrement by amount
err := db.Query().Where("id", 1).Decrement("stock", 2)

// Raw SQL
db.Query().Raw("SELECT * FROM users WHERE created_at > ?", time.Now().AddDate(-7, 0, 0))

// Execute raw SQL
result, err := db.Query().Exec("UPDATE users SET active = ? WHERE id = ?", true, 1)

// Omit columns from INSERT/UPDATE
user := User{Name: "John", Email: "john@example.com", InternalField: "secret"}
err := db.Query().Omit("internal_field").Create(&user)

// Lock for update
err := db.Query().Where("id", 1).LockForUpdate().First(&user)

// Shared lock
err := db.Query().Where("id", 1).SharedLock().First(&user)

// Apply scopes
func ActiveUsers(q Query) Query {
    return q.Where("active", true)
}
func PremiumUsers(q Query) Query {
    return q.Where("plan", "premium")
}
err := db.Query().Scopes(ActiveUsers, PremiumUsers).Get(&users)

Configuration Methods

// Specify table name
db.Query().Table("users").Get(&users)

// Set model for query
db.Query().Model(&User{}).Where("active", true).Get(&users)

// Use specific connection
db.Query().Connection("read_replica").Get(&users)

// Set context
ctx := context.WithTimeout(context.Background(), 5*time.Second)
db.Query().WithContext(ctx).Get(&users)

// Clone query
query1 := db.Query().Where("active", true)
query2 := query1.Clone().Where("premium", true)

// Get database connection
dbConn := db.Query().DB()

// Get read database connection
readConn := db.Query().ReadDB()

// Get database driver
driver := db.Query().Driver()

// Register observer
db.Query().Observe(&User{}, &UserObserver{})

// Disable events
db.Query().WithoutEvents().Create(&user)

Debug and Logging Methods

// Enable debug mode
db.Query().EnableDebug().Get(&users)

// Disable debug mode
db.Query().DisableDebug().Get(&users)

// Check if debug enabled
if db.Query().IsDebug() {
    fmt.Println("Debug mode is on")
}

// Enable query logging
db.Query().EnableQueryLog().Get(&users)

// Disable query logging
db.Query().DisableQueryLog().Get(&users)

// Flush query log
db.Query().FlushQueryLog()

// Get query log
logs := db.Query().GetQueryLog()
for _, log := range logs {
    fmt.Printf("Query: %s\n", log.Sql)
    fmt.Printf("Duration: %v\n", log.Duration)
}

Advanced Queries

Joins

// Inner join
db.Query().Join("posts", "users.id", "=", "posts.user_id")

// Left join
db.Query().LeftJoin("posts", "users.id", "=", "posts.user_id")

Group By and Having

db.Query().GroupBy("category").Having("count", ">", 10)

Subqueries

// Subquery in where clause
subquery := db.Query().Select("id").From("orders").Where("status", "pending")
db.Query().WhereIn("user_id", subquery)

Creating, Updating, Deleting

Create

user := User{Name: "John", Email: "john@example.com"}
err := db.Query().Create(&user)

Update

// Update single column
err := db.Query().Where("id", 1).Update("name", "Jane")

// Update multiple columns
err := db.Query().Where("id", 1).Update(map[string]any{
    "name": "Jane",
    "email": "jane@example.com",
})

Delete

// Delete records
result, err := db.Query().Where("id", 1).Delete()

// Force delete (with soft deletes)
result, err := db.Query().Where("id", 1).ForceDelete()

Method Chaining

All query builder methods can be chained for fluent interface:

var users []User
err := db.Query().
    Where("age", ">", 18).
    Where("active", true).
    OrderBy("created_at", "desc").
    Limit(10).
    Get(&users)
