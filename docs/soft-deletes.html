Soft Deletes

Soft deleting a record doesn't actually remove it from your database. Instead, it sets a timestamp on the record indicating when it was "deleted". Neat ORM provides multiple strategies to handle soft deletes, giving you flexibility based on your database schema requirements.

Basic Usage

To enable soft deletes for a model, embed one of the soft delete structs from the github.com/dracory/neat/database/soft_delete package.

type User struct {
    soft_delete.SoftDeletes // Embed soft delete functionality
    ID   uint
    Name string
}

Soft Delete Strategies

Neat ORM supports two primary strategies for soft deletes:

1. NULL-Based Strategy (Default)

The record is considered active when the timestamp column is NULL, and deleted when it contains a timestamp.

  • SoftDeletes: Uses soft_deleted_at column (Recommended).
  • DeletedAt: Uses deleted_at column (Laravel-compatible).
// Example
type User struct {
    soft_delete.SoftDeletes
    ID uint
}

2. Max-Date Sentinel Strategy

Instead of NULL, this strategy uses a far-future date (9999-12-31) as a "not deleted" sentinel. This is useful for databases or schemas that enforce NOT NULL constraints on timestamp columns or for better indexing performance in some environments.

  • SoftDeletesMaxDate: Uses soft_deleted_at with sentinel.
  • DeletedAtMaxDate: Uses deleted_at with sentinel.
// Example
type User struct {
    soft_delete.SoftDeletesMaxDate
    ID uint
}

Querying Soft Deleted Records

By default, soft-deleted records are excluded from all queries.

Include Soft Deleted Records

var users []User
db.Query().WithSoftDeleted().Get(&users)

Only Soft Deleted Records

var users []User
db.Query().OnlySoftDeleted().Get(&users)

Performing Operations

Soft Delete a Record

// Via Query Builder
db.Query().Where("id", 1).Delete()

// Via Model
user.SoftDelete()
db.Query().Save(&user)

Restore a Soft Deleted Record

// Via Query Builder
db.Query().Where("id", 1).RestoreSoftDeleted()

// Via Model
user.RestoreSoftDeleted()
db.Query().Save(&user)

Permanent Delete (Force Delete)

To permanently remove a soft-deletable model from the database:

db.Query().Where("id", 1).ForceDelete()

Schema Integration

When creating your table with the Schema Builder, make sure to add the appropriate column:

// For NULL-based (default)
db.Schema().Create("users", func(table contractsschema.Blueprint) {
    table.ID()
    table.String("name")
    table.SoftDeletes() // Adds nullable soft_deleted_at
})

// For Max-Date sentinel
db.Schema().Create("users", func(table contractsschema.Blueprint) {
    table.ID()
    table.String("name")
    // Adds NOT NULL column with far-future default
    table.Timestamp("soft_deleted_at").Default("9999-12-31 23:59:59")
})
