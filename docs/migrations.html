Migrations New: migrator Package

Neat ORM features a powerful and elegant migration system through the database/migrator package. Migrations allow you to version-control your database schema, making it easy for your team to modify and share the application's database schema.

🚀 Key Advantage: Most Go ORMs lack comprehensive schema migration support. Neat ORM includes a complete migration system with schema builder, rollback support, and automatic tracking.

Overview

The migrator package provides a clean API for managing database migrations with:

  • Automatic Schema Injection: No need to manually register migrations with schema.
  • Clean Interface-based API: Uses MigratorInterface for consistent operations.
  • Context Support: All operations support context.Context for cancellation and timeouts.
  • Automatic Tracking: Automatically tracks executed migrations in a migration_tracker table.
  • Flexible Rollback: Support for rolling back by specific steps or to a specific batch.

Migration Implementation

Migrations must implement the MigrationInterface. Typically, you embed schema.BaseMigration for convenience:

import (
    contractsschema "github.com/dracory/neat/contracts/database/schema"
    "github.com/dracory/neat/database/schema"
)

type CreateUsersTable struct {
    schema.BaseMigration
}

func (m *CreateUsersTable) Signature() string {
    return "2024_06_15_120000_create_users_table"
}

func (m *CreateUsersTable) Description() string {
    return "Creates users table"
}

func (m *CreateUsersTable) Up() error {
    return m.GetSchema().Create("users", func(blueprint contractsschema.Blueprint) {
        blueprint.ID()
        blueprint.String("name")
        blueprint.String("email").Unique()
        blueprint.Timestamps()
    })
}

func (m *CreateUsersTable) Down() error {
    return m.GetSchema().DropIfExists("users")
}

Using the migrator

The migrator is the entry point for running your migrations. It requires a neat.Database instance.

import (
    "context"
    "github.com/dracory/neat"
    "github.com/dracory/neat/database/migrator"
)

func main() {
    db, _ := neat.NewFromDSN("sqlite://./app.db")
    defer db.Close()

    // Create migrator instance
    s := migrator.NewMigrator(db)

    // Add migrations
    s.AddMigrations([]contractsschema.MigrationInterface{
        &CreateUsersTable{},
        &CreatePostsTable{},
    })

    // Run pending migrations
    ctx := context.Background()
    if err := s.Up(ctx); err != nil {
        log.Fatal(err)
    }
}

Available Operations

Run All Pending Migrations

err := s.Up(ctx)

Rollback Last Migration

err := s.Down(ctx)

Rollback Multiple Steps

// Rollback the last 3 migrations
err := s.RollbackSteps(ctx, 3)

Rollback to Specific Batch

err := s.RollbackToBatch(ctx, 20240615120000)

Check Migration Status

status, err := s.Status()
for _, s := range status {
    fmt.Printf("Migration: %s - State: %s\n", s.ID, s.State)
}

Reset All Migrations

Rolls back every migration that has ever been run.

err := s.Reset(ctx)

Fresh Migration

Drops all tables and re-runs all migrations. Useful for development.

err := s.Fresh(ctx)

Advanced Configuration

Transaction Wrapping

By default, migrator wraps migration operations in a database transaction for safety. You can disable this if needed (e.g., for DDL operations that don't support transactions):

s.SetTransactionsEnabled(false)

Custom Tracking Table

The default table name for tracking migrations is migration_tracker. You can change this:

s.SetTableName("my_migrations")

Signature Validation

You can enforce specific naming conventions for your migration signatures:

s.SetSignatureValidation(true, migrator.SignatureFormatDateTime)
