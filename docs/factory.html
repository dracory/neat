Factory Pattern

The Factory pattern in Neat ORM provides a convenient way to create test data for your database. It allows you to define model templates and generate multiple instances with optional attribute overrides.

Overview

The Factory system consists of:

  • Factory Interface: Defines the contract for creating model instances
  • Factory Implementation: Provides methods for creating and persisting models
  • Model Factory: Optional interface for models to define their own factory

Basic Usage

Creating a Single Model

// Create a single user in the database
user := &User{Name: "John", Email: "john@example.com"}
createdUser, err := db.Factory().Table("users").Create(user)

Creating Multiple Models

// Create 5 users in the database
template := &User{Name: "Template User"}
users, err := db.Factory().Table("users").Count(5).Create(template)

Creating Models with Custom Attributes

// Create a user with custom attributes
template := &User{Name: "Template"}
user, err := db.Factory().Table("users").Create(template, map[string]any{
    "Name":  "Custom Name",
    "Email": "custom@example.com",
})

Creating In-Memory Models

// Create a user without persisting to database
user, err := db.Factory().Make(&User{Name: "Test User"})

Creating Multiple In-Memory Models

// Create 3 users in memory
users, err := db.Factory().Count(3).Make(&User{Name: "Template"})

Factory Methods

Table(table string) Factory

Sets the table name for database operations. This is required for Create() and CreateQuietly() operations.

factory := db.Factory().Table("users")

Count(count int) Factory

Sets the number of models that should be generated. Can be chained with Create, CreateQuietly, or Make for bulk operations.

factory := db.Factory().Count(10)

Create(value any, attributes ...map[string]any) (any, error)

Creates a model and persists it to the database, returning the created instance(s). Requires Table() to be called first.

Make(value any, attributes ...map[string]any) (any, error)

Creates a model in memory without persisting to the database. Useful for test setup where you don't need to hit the database.

CreateQuietly(value any, attributes ...map[string]any) (any, error)

Creates a model and persists it to database without firing observer events. Useful for seeding data where you don't want to trigger side effects like emails.

// Create without triggering observers
user, err := db.Factory().Table("users").CreateQuietly(&User{
    Name: "System User",
    Email: "system@example.com",
})

Factory States

Define different states for your models to represent different scenarios:

type UserFactory struct{}

func (f *UserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        States: map[string]map[string]any{
            "admin": {
                "is_admin": true,
                "role": "admin",
            },
            "premium": {
                "is_premium": true,
                "plan": "premium",
            },
            "inactive": {
                "active": false,
            },
        },
    }
}

// Use states
factory := db.Factory().Model(&UserFactory{})
adminUser, _ := factory.State("admin").Create()
premiumUser, _ := factory.State("premium").Create()

Factory Sequences

Use sequences to generate unique values across multiple factory calls:

type UserFactory struct{}

func (f *UserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        Sequences: map[string]func(int) any{
            "email": func(n int) any {
                return fmt.Sprintf("user%d@example.com", n)
            },
        },
    }
}

// Each call will generate a unique email
user1, _ := factory.Create() // user1@example.com
user2, _ := factory.Create() // user2@example.com
user3, _ := factory.Create() // user3@example.com

Factory Relationships

Create related models using factory associations:

// Create users with posts
factory := db.Factory().Table("users")

for i := 0; i < 10; i++ {
    user, _ := factory.Create(&User{
        Name: fmt.Sprintf("User %d", i),
    })
    
    // Create posts for each user
    postFactory := db.Factory().Table("posts")
    for j := 0; j < 5; j++ {
        postFactory.Create(&Post{
            UserID: user.ID,
            Title:  fmt.Sprintf("Post %d", j),
        })
    }
}

Factory Callbacks

Define callbacks to modify models before/after creation:

type UserFactory struct{}

func (f *UserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        AfterCreating: func(model any) error {
            user := model.(*User)
            // Hash password after creation
            user.Password = hashPassword(user.Password)
            return nil
        },
    }
}

Factory Traits

Define reusable traits that can be applied to multiple factories:

type HasProfileTrait struct{}

func (t *HasProfileTrait) Apply(model any) {
    user := model.(*User)
    user.Profile = &Profile{
        Bio: "Default bio",
    }
}

type UserFactory struct{}

func (f *UserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        Traits: []Trait{HasProfileTrait{}},
    }
}

Factory with Transactions

Wrap factory operations in transactions for data consistency:

err := db.Query().Transaction(func(tx Query) error {
    factory := tx.Factory().Table("users")
    
    // All operations in transaction
    for i := 0; i < 10; i++ {
        if _, err := factory.Create(&User{
            Name: fmt.Sprintf("User %d", i),
        }); err != nil {
            return err // Will rollback
        }
    }
    
    return nil // Will commit
})

Factory with Observers

By default, factory operations trigger observer events. Use CreateQuietly() to disable:

// This will trigger Creating and Created events
user, err := db.Factory().Table("users").Create(&User{
    Name: "John",
})

// This will NOT trigger observer events
user, err := db.Factory().Table("users").CreateQuietly(&User{
    Name: "John",
})

Factory Validation

Implement validation in factory callbacks:

type UserFactory struct{}

func (f *UserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        BeforeCreating: func(model any) error {
            user := model.(*User)
            
            if user.Name == "" {
                return errors.New("name is required")
            }
            
            if !isValidEmail(user.Email) {
                return errors.New("invalid email")
            }
            
            return nil
        },
    }
}

Factory Error Handling

Handle factory creation errors appropriately:

users, err := db.Factory().Table("users").Count(10).Create(&User{
    Name: "Template User",
})

if err != nil {
    // Handle creation error
    log.Printf("Failed to create users: %v", err)
    return err
}

// Continue with created users
for _, user := range users.([]User) {
    fmt.Printf("Created user: %s\n", user.Name)
}

Factory Batch Operations

Create large batches efficiently:

// Create 1000 users in batches of 100
batchSize := 100
total := 1000

for i := 0; i < total; i += batchSize {
    end := i + batchSize
    if end > total {
        end = total
    }
    
    _, err := db.Factory().Table("users").Count(end - i).Create(&User{
        Name: "Batch User",
    })
    
    if err != nil {
        log.Printf("Failed to create batch %d: %v", i/batchSize, err)
        return err
    }
}

Factory Custom Attributes

Override template attributes with custom values:

template := &User{
    Name:  "Default Name",
    Email: "default@example.com",
}

// Override specific attributes
user, err := db.Factory().Table("users").Create(template, map[string]any{
    "Name":  "Custom Name",
    "Email": "custom@example.com",
    "Age":   25,
})

Factory Inheritance

Extend base factories for specific use cases:

type BaseUserFactory struct{}

func (f *BaseUserFactory) Definition() FactoryDefinition {
    return FactoryDefinition{
        Model: &User{},
        Attributes: map[string]any{
            "active": true,
            "verified": false,
        },
    }
}

type AdminUserFactory struct {
    BaseUserFactory
}

func (f *AdminUserFactory) Definition() FactoryDefinition {
    def := f.BaseUserFactory.Definition()
    def.Attributes["is_admin"] = true
    def.Attributes["role"] = "admin"
    return def
}

Practical Example

// Create test users with varying data
for i := 0; i < 100; i++ {
    template := &User{
        Name:  faker.Name(),
        Email: faker.Email(),
    }
    
    // 10% chance of admin
    isAdmin := rand.Float32() < 0.1
    
    _, err := db.Factory().Table("users").Create(template, map[string]any{
        "IsAdmin": isAdmin,
    })
    if err != nil {
        log.Fatal(err)
    }
}
