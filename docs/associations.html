Associations

This document describes the model associations (relationships) in Neat ORM.

Supported Relationships

  • BelongsTo
  • HasMany
  • HasOne
  • PolymorphicBelongsTo
  • PolymorphicHasMany

BelongsTo

A BelongsTo relationship defines a many-to-one relationship where a model belongs to another model.

type Post struct {
    ID     uint
    Title  string
    UserID uint
    User   User // BelongsTo
}

type User struct {
    ID    uint
    Name  string
    Posts []Post // HasMany
}

Querying BelongsTo

// Eager load the relationship
db.Query().With("user").Where("id", 1).First(&post)

// Lazy load the relationship
db.Query().Load(&post, "user")

HasMany

A HasMany relationship defines a one-to-many relationship where a model has many related models.

type User struct {
    ID    uint
    Name  string
    Posts []Post // HasMany
}

type Post struct {
    ID     uint
    Title  string
    UserID uint
    User   User // BelongsTo
}

Querying HasMany

// Eager load the relationship
db.Query().With("posts").Where("id", 1).First(&user)

// Lazy load the relationship
db.Query().Load(&user, "posts")

HasOne

A HasOne relationship defines a one-to-one relationship where a model has one related model.

type User struct {
    ID      uint
    Name    string
    Profile Profile // HasOne
}

type Profile struct {
    ID     uint
    UserID uint
    Bio    string
    User   User // BelongsTo
}

Querying HasOne

// Eager load the relationship
db.Query().With("profile").Where("id", 1).First(&user)

// Lazy load the relationship
db.Query().Load(&user, "profile")

Eager Loading

Eager loading allows you to load relationships with the main model to avoid N+1 query problems:

// Single relationship
db.Query().With("posts").Where("id", 1).First(&user)

// Multiple relationships
db.Query().With("posts").With("profile").Where("id", 1).First(&user)

// Nested relationships
db.Query().With("posts.comments").Where("id", 1).First(&user)

Lazy Loading

Lazy loading allows you to load relationships on demand:

// Load relationship after fetching the model
db.Query().First(&user, 1)
db.Query().Load(&user, "posts")

// Load multiple relationships
db.Query().Load(&user, "posts", "profile")

Association Operations

Append

// Add a related model
db.Query().Association("posts").Append(&user, &post)

Replace

// Replace all related models
db.Query().Association("posts").Replace(&user, &newPosts)

Delete

// Remove a related model
db.Query().Association("posts").Delete(&user, &post)

Clear

// Remove all related models
db.Query().Association("posts").Clear(&user)

Count

// Count related models
count := db.Query().Association("posts").Count(&user)

Polymorphic Relationships

Polymorphic relationships allow a model to belong to multiple other models on a single association.

PolymorphicBelongsTo

type Comment struct {
    ID            uint
    Body          string
    CommentableID uint
    CommentableType string
    Commentable   polymorphic.Commentable // PolymorphicBelongsTo
}

// Can belong to Post or User
type Post struct {
    ID       uint
    Title    string
    Comments []Comment `neat:"polymorphic:Commentable"`
}

type User struct {
    ID       uint
    Name     string
    Comments []Comment `neat:"polymorphic:Commentable"`
}

PolymorphicHasMany

// Post has many comments (polymorphic)
type Post struct {
    ID       uint
    Title    string
    Comments []Comment `neat:"polymorphic:Commentable"`
}
