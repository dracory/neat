API Reference

This document provides a comprehensive reference for the Neat ORM API. For detailed usage examples and best practices, see the specific documentation for each component.

Database

Configuration

type DBConfig struct {
    Default     string
    Connections map[string]ConnectionConfig
    Migrations  MigrationConfig
}

type ConnectionConfig struct {
    Driver   string
    Host     string
    Port     int
    Database string
    Username string
    Password string
    Options  map[string]string
    Pool     PoolConfig
}

type PoolConfig struct {
    MaxOpenConns int
    MaxIdleConns int
    MaxLifetime  int
}

Methods

  • New(config DBConfig) (*Database, error) - Create database from config
  • NewFromDSN(dsn string) (*Database, error) - Create database from DSN string
  • Close() error - Close database connection
  • Query() Query - Get query builder instance
  • Schema() Schema - Get schema builder instance
  • Orm() Orm - Get ORM instance
  • Factory() Factory - Get factory instance
  • Transaction(fn func(Query) error) error - Execute transaction
  • Migrate(paths ...string) error - Run migrations
  • Seed(seeders []Seeder) error - Run seeders

Query Builder

Selection Methods

  • Select(query any, args ...any) Query - Select specific columns (supports subqueries, raw expressions)
  • Distinct(args ...any) Query - Return distinct rows
  • Pluck(column string, dest any) error - Retrieve single column values
  • Value(column string, dest any) error - Retrieve single value from first row

Retrieval Methods

  • Get(dest any) error - Retrieve all rows
  • All(dest any) error - Alias for Get (Django-style)
  • FindAll(dest any) error - Alias for All (Sequelize-style)
  • First(dest any) error - Retrieve first row
  • FindOne(dest any) error - Alias for First (Sequelize-style)
  • Find(dest any, conds ...any) error - Retrieve rows with optional conditions
  • FirstOrFail(dest any) error - First or error
  • FindOrFail(dest any, conds ...any) error - Find or error
  • FirstOr(dest any, callback func() error) error - First or execute callback
  • FirstOrCreate(dest any, conds ...any) error - First or create
  • FirstOrNew(dest any, attributes any, values ...any) error - First or new instance
  • Scan(dest any) error - Scan raw query results

Where Clauses

  • Where(query any, args ...any) Query - Add WHERE clause (Laravel-style)
  • Filter(query any, args ...any) Query - Alias for Where (Django-style)
  • OrWhere(query any, args ...any) Query - Add OR WHERE clause
  • WhereIn(column string, values []any) Query - WHERE column IN (...)
  • WhereNotIn(column string, values []any) Query - WHERE column NOT IN (...)
  • OrWhereIn(column string, values []any) Query - OR WHERE column IN (...)
  • OrWhereNotIn(column string, values []any) Query - OR WHERE column NOT IN (...)
  • WhereBetween(column string, x, y any) Query - WHERE column BETWEEN x AND y
  • WhereNotBetween(column string, x, y any) Query - WHERE column NOT BETWEEN x AND y
  • OrWhereBetween(column string, x, y any) Query - OR WHERE column BETWEEN x AND y
  • OrWhereNotBetween(column string, x, y any) Query - OR WHERE column NOT BETWEEN x AND y
  • WhereNull(column string) Query - WHERE column IS NULL
  • WhereNotNull(column string) Query - WHERE column IS NOT NULL
  • OrWhereNull(column string) Query - OR WHERE column IS NULL
  • WhereColumn(first, operator, second string) Query - WHERE column1 operator column2
  • OrWhereColumn(first, operator, second string) Query - OR WHERE column1 operator column2
  • WhereExists(callback func(orm.Query) orm.Query) Query - WHERE EXISTS (subquery)
  • WhereNot(query any, args ...any) Query - WHERE NOT (condition)
  • Exclude(query any, args ...any) Query - Alias for WhereNot (Django-style)
  • OrWhereNot(query any, args ...any) Query - OR WHERE NOT (condition)
  • WhereAny(columns []string, operator string, value any) Query - WHERE (col1 op ? OR col2 op ? OR ...)
  • WhereAll(columns []string, operator string, value any) Query - WHERE (col1 op ? AND col2 op ? AND ...)
  • WhereNone(columns []string, operator string, value any) Query - WHERE NOT (col1 op ? OR col2 op ? OR ...)

JSON Where Methods

  • WhereJsonContains(column string, value any) Query - WHERE JSON column contains value
  • OrWhereJsonContains(column string, value any) Query - OR WHERE JSON column contains value
  • WhereJsonDoesntContain(column string, value any) Query - WHERE JSON column doesn't contain value
  • OrWhereJsonDoesntContain(column string, value any) Query - OR WHERE JSON column doesn't contain value
  • WhereJsonContainsKey(column string) Query - WHERE JSON column contains key
  • OrWhereJsonContainsKey(column string) Query - OR WHERE JSON column contains key
  • WhereJsonDoesntContainKey(column string) Query - WHERE JSON column doesn't contain key
  • OrWhereJsonDoesntContainKey(column string) Query - OR WHERE JSON column doesn't contain key
  • WhereJsonLength(column string, operator string, value any) Query - WHERE JSON column length operator value

  • OrderByDesc(column string) Query - ORDER BY column DESC

Join Methods

  • LeftJoin(query string, args ...any) Query - LEFT JOIN
  • RightJoin(query string, args ...any) Query - RIGHT JOIN

Create/Insert Methods

  • Create(value any) error - Insert new record

Update Methods

Delete Methods

Soft Delete Embeds

Neat provides multiple soft delete embeds in the github.com/dracory/neat/database/soft_delete package. For detailed usage examples and strategy comparisons, see the Soft Deletes documentation.

NULL-Based Strategy

  • SoftDeletes - Uses soft_deleted_at column (NULL = active)
  • SoftDeletedAt - Uses soft_deleted_at column (explicit naming)
  • DeletedAt - Uses deleted_at column (Laravel-compatible)

Max-Date Sentinel Strategy

  • SoftDeletesMaxDate - Uses soft_deleted_at with NOT NULL support
  • DeletedAtMaxDate - Uses deleted_at with NOT NULL support

Model Methods

When using soft delete embeds, models gain these methods:

  • IsSoftDeleted() bool - Check if record is soft deleted
  • SoftDelete() - Mark record as soft deleted (in-memory)
  • RestoreSoftDeleted() - Mark record as not soft deleted (in-memory)
  • SoftDeletedAtColumn() string - Get the soft delete column name
  • GetSoftDeletedAt() *time.Time - Get the soft delete timestamp

Soft Delete Column Setup

For NULL-based strategy:

blueprint.Timestamp(constants.SoftDeleteAtColumn).Nullable()
// or
blueprint.Timestamp(constants.DeletedAtColumnName).Nullable()

For Max-Date sentinel strategy:

blueprint.Timestamp(constants.SoftDeleteAtColumn).Default(constants.MaxSoftDeletedAtDefault)
// or
blueprint.Timestamp(constants.DeletedAtColumnName).Default(constants.MaxSoftDeletedAtDefault)
