Observers

This document describes the observer system in Neat ORM for reacting to model lifecycle events.

What are Observers?

Observers allow you to hook into model lifecycle events to perform actions before or after database operations.

Observer Events

Creating

Called before a model is created:

func (o *UserObserver) Creating(event neat.Event) error {
    // Validate or modify data before creation
    return nil
}

Created

Called after a model is created:

func (o *UserObserver) Created(event neat.Event) error {
    // Perform post-creation actions (e.g., send email)
    return nil
}

Updating

Called before a model is updated:

func (o *UserObserver) Updating(event neat.Event) error {
    // Validate or modify data before update
    return nil
}

Updated

Called after a model is updated:

func (o *UserObserver) Updated(event neat.Event) error {
    // Perform post-update actions
    return nil
}

Deleting

Called before a model is deleted:

func (o *UserObserver) Deleting(event neat.Event) error {
    // Perform pre-deletion actions
    return nil
}

Deleted

Called after a model is deleted:

func (o *UserObserver) Deleted(event neat.Event) error {
    // Perform post-deletion actions
    return nil
}

Force Deleted

Called after a model is force deleted (permanently deleted):

func (o *UserObserver) ForceDeleted(event neat.Event) error {
    // Perform post-force-deletion actions
    return nil
}

Restoring

Called before a soft-deleted model is restored:

func (o *UserObserver) Restoring(event neat.Event) error {
    // Validate or modify data before restore
    return nil
}

Restored

Called after a soft-deleted model is restored:

func (o *UserObserver) Restored(event neat.Event) error {
    // Perform post-restore actions
    return nil
}

Saving

Called before a model is saved (create or update):

func (o *UserObserver) Saving(event neat.Event) error {
    // Validate or modify data before save
    return nil
}

Saved

Called after a model is saved (create or update):

func (o *UserObserver) Saved(event neat.Event) error {
    // Perform post-save actions
    return nil
}

Retrieved

Called after a model is retrieved from the database:

func (o *UserObserver) Retrieved(event neat.Event) error {
    // Perform post-retrieval actions (e.g., load relationships)
    return nil
}

Event Lifecycle

Observer events follow this lifecycle:

  • Create: Creating → Created
  • Update: Saving → Updating → Updated → Saved
  • Delete: Deleting → Deleted (or ForceDeleted)
  • Restore: Restoring → Restored
  • Retrieve: Retrieved

Event Propagation Control

Return an error from any event method to stop the operation:

func (o *UserObserver) Creating(event neat.Event) error {
    user := event.GetModel().(*User)
    
    // Return error to prevent creation
    if user.Email == "" {
        return errors.New("email is required")
    }
    
    return nil // Continue with creation
}

Event Context

Access the event context for additional information:

func (o *UserObserver) Created(event neat.Event) error {
    ctx := event.GetContext()
    
    // Access context values
    if userID := ctx.Value("user_id"); userID != nil {
        log.Printf("Created by user: %v", userID)
    }
    
    return nil
}

Multiple Observers

You can register multiple observers for the same model. They will be called in registration order:

db.Orm().Observe([]neat.ModelToObserver{
    {Model: User{}, Observer: ValidationObserver{}},
    {Model: User{}, Observer: LoggingObserver{}},
    {Model: User{}, Observer: NotificationObserver{}},
})

Observer Registration Options

// Register with query builder
db.Query().Observe(&User{}, &UserObserver{})

// Register with ORM
db.Orm().Observe(&User{}, &UserObserver{})

Observer Error Handling

Errors returned from observer methods will stop the operation and return the error to the caller:

func (o *UserObserver) Creating(event neat.Event) error {
    user := event.GetModel().(*User)
    
    if err := validateUser(user); err != nil {
        return err // This will prevent the creation
    }
    
    return nil
}

Observer Priorities

Observers are called in the order they are registered. Register critical observers first:

db.Orm().Observe([]neat.ModelToObserver{
    {Model: User{}, Observer: SecurityObserver{}},      // Called first
    {Model: User{}, Observer: ValidationObserver{}},    // Called second
    {Model: User{}, Observer: LoggingObserver{}},      // Called third
})

Observer Lifecycle Hooks

Observers can maintain state across multiple events:

type AuditObserver struct {
    mu       sync.Mutex
    changes  map[uint][]string
}

func (o *AuditObserver) Updating(event neat.Event) error {
    o.mu.Lock()
    defer o.mu.Unlock()
    
    user := event.GetModel().(*User)
    o.changes[user.ID] = append(o.changes[user.ID], "user_updated")
    
    return nil
}

func (o *AuditObserver) Deleted(event neat.Event) error {
    o.mu.Lock()
    defer o.mu.Unlock()
    
    user := event.GetModel().(*User)
    o.changes[user.ID] = append(o.changes[user.ID], "user_deleted")
    
    return nil
}

Observer Transaction Integration

Observers work seamlessly with transactions. If a transaction fails, observer events will not be called for the failed operation:

err := db.Query().Transaction(func(tx Query) error {
    // Creating events will fire
    if err := tx.Create(&user); err != nil {
        return err // Created event will not fire
    }
    
    // Created event fires here
    
    // Updating events will fire
    if err := tx.Update(&user); err != nil {
        return err // Updated event will not fire
    }
    
    // Updated event fires here
    
    return nil
})

Disabling Observers

Temporarily disable observers for specific operations:

// Disable observers for this query
db.Query().WithoutEvents().Create(&user)

Observer Event Filtering

You can conditionally execute observer logic based on event context:

func (o *UserObserver) Creating(event neat.Event) error {
    ctx := event.GetContext()
    
    // Skip validation for system users
    if ctx.Value("skip_validation") == true {
        return nil
    }
    
    user := event.GetModel().(*User)
    return validateUser(user)
}

Observer Batch Operations

Observers are called for each model in batch operations:

// Each Create will fire Creating and Created events
users := []User{{Name: "John"}, {Name: "Jane"}}
for _, user := range users {
    db.Query().Create(&user)
}

Creating an Observer

type UserObserver struct{}

func (o *UserObserver) Creating(event neat.Event) error {
    log.Println("Creating user")
    return nil
}

func (o *UserObserver) Created(event neat.Event) error {
    log.Println("User created")
    return nil
}

Registering Observers

// Register observers with the ORM
db.Orm().Observe([]neat.ModelToObserver{
    {Model: User{}, Observer: UserObserver{}},
    {Model: Post{}, Observer: PostObserver{}},
})

Event Interface

The event interface provides access to the model being operated on:

type Event interface {
    GetModel() any
    GetContext() context.Context
}

Practical Example

type UserObserver struct{}

func (o *UserObserver) Creating(event neat.Event) error {
    user := event.GetModel().(*User)
    
    // Validate email format
    if !isValidEmail(user.Email) {
        return errors.New("invalid email format")
    }
    
    // Hash password before saving
    if user.Password != "" {
        hashed, err := bcrypt.GenerateFromPassword([]byte(user.Password), 10)
        if err != nil {
            return err
        }
        user.Password = string(hashed)
    }
    
    return nil
}

func (o *UserObserver) Created(event neat.Event) error {
    user := event.GetModel().(*User)
    
    // Send welcome email
    go sendWelcomeEmail(user.Email, user.Name)
    
    return nil
}

func (o *UserObserver) Deleting(event neat.Event) error {
    user := event.GetModel().(*User)
    
    // Prevent deletion of admin users
    if user.IsAdmin {
        return errors.New("cannot delete admin users")
    }
    
    return nil
}
