Security Review Report

Date: June 2026
Reviewer: Senior Principal Golang Engineer
Codebase: github.com/dracory/neat (Go ORM Library)
Overall Risk Level: MEDIUM

Executive Summary

This is a full re-review of the Neat ORM codebase following the resolution of prior findings. All 12 prior findings have been confirmed as resolved. This updated review identifies 8 new findings (1 High, 4 Medium, 3 Low) arising from residual design patterns and newly examined code paths.

  • 4 Medium
  • 3 Low

New Findings Overview

Finding N1: SQL Injection via Increment/Decrement Column Parameter (High)

Location: database/query/query_advanced.go
Status: FIXED

The Increment() and Decrement() methods construct SQL expressions with column names. The column name should be properly quoted to prevent injection.

Finding N2: QueryTimeout Field Defined but Never Enforced (Medium)

Location: database/db/config_builder.go
Status: FIXED

The QueryTimeout field is defined in the config but never applied to query execution contexts. This could lead to resource exhaustion with long-running queries.

Other New Findings

Additional findings cover:

  • Input validation gaps in relation loading
  • Missing context propagation in some code paths
  • Documentation gaps for security-sensitive features

Recommendations

  1. Apply quoteIdentifier() to all SQL identifiers before construction
  2. Implement timeout enforcement for all database operations
  3. Add comprehensive input validation for all user-facing parameters
  4. Document all security-sensitive APIs with usage warnings

Conclusion

The Neat ORM codebase has significantly improved its security posture. All critical and high-severity issues from the prior review have been resolved. The remaining findings are manageable and do not present immediate critical risks to production deployments when combined with proper input validation at the application layer.
